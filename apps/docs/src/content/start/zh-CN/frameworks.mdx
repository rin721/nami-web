# 框架接入

Nami UI 使用标准 Web Components，不需要 React、Vue 或 Astro wrapper。应用只需要注册组件、加载主题 CSS，然后通过 attributes、properties 和 DOM events 传递状态。

## 原生 HTML

```html
<script type="module">
  import '@nami/themes/default.css';
  import '@nami/ui/register';
</script>

<nami-button>保存</nami-button>
```

## Vue

```ts
import '@nami/themes/default.css';
import '@nami/ui/register';
```

在 Vue compiler options 中把 `nami-*` 配置为 custom elements。事件可以用原生 DOM 事件监听，复杂表单值可以通过 refs 设置 property。

## React

```tsx
import '@nami/themes/default.css';
import '@nami/ui/register';

export function App() {
  return <nami-button>Save</nami-button>;
}
```

需要类型化事件时，可以通过 refs 监听，也可以在项目内写一个很小的事件桥接 wrapper。

## 页面过渡钩子

路由顶部进度使用 `nami-top-progress`；首屏揭幕和阻断式任务使用 `nami-page-transition`。两者都是标准元素，框架只需要在自己的生命周期里调用方法。

### Astro

```ts
const progress = document.querySelector('nami-top-progress');

document.addEventListener('astro:before-preparation', () => {
  progress?.show({ progress: 12 });
});

document.addEventListener('astro:after-preparation', () => {
  progress?.finish();
});
```

### Vue Router

```ts
const progress = document.querySelector('nami-top-progress');

router.beforeResolve(() => {
  progress?.show({ progress: 16 });
});

router.afterEach(() => {
  progress?.finish();
});
```

### 原生异步任务

```ts
const transition = document.querySelector('nami-page-transition');

await transition?.waitFor(fetch('/api/profile'), {
  appearance: 'panel',
  minDuration: 240
});
```

推荐路由切换使用 `nami-top-progress`，首屏初始化使用 `appearance="veil"`，只有真正阻断交互的任务才使用 `appearance="panel"`。
