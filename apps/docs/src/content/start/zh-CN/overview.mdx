# 总览

Nami UI 是一套 Web Components UI 组件库，适合需要在多个前端技术栈中复用同一套组件源码的应用。公共边界是自定义元素、反射属性、DOM 事件、slots、CSS Parts 和 CSS Variables。

## 这里放什么

Start 只负责理解项目定位、安装方式、注册方式、单组件导入和框架接入。它是第一次使用 Nami UI 的路径，不承载完整组件 API，也不展开主题算法细节。

## 不放什么

组件属性、事件、slots、parts 和 tokens 归入 Components；主题配置、tokens、算法、风格 preset、Theme Designer 和 Theme Lab 归入 Theme；质量、i18n 和语义结构契约归入 Resources。

## 下一步去哪

已经准备接入项目时，继续阅读 [快速开始](/zh-CN/docs/getting-started/)；需要选组件时进入 [组件目录](/zh-CN/components/)；需要做品牌主题时进入 [主题系统](/zh-CN/theme/)。

## 第一阶段包含什么

| 模块 | 含义 |
| --- | --- |
| 组件 | Button、Chip、Input、Switch、Card、Tab、浮层、应用壳和状态反馈 |
| 主题 | 浅色、暗色、自定义主题色、紧凑密度、减少动效 |
| 皮肤 | 默认社区风格，以及同一套 token 驱动的插画风格 |
| i18n | `@lit/localize` runtime mode、语言前缀路由和组件 fallback 文案 |
