# 总览

Nami UI 是基于 Lit 与 Web Components 的组件库。它把组件行为、主题 token、CSS Parts、语义结构和元数据放在同一个工程契约里，因此可以在原生 HTML、Vue、React、Astro 或任何支持 custom elements 的环境中使用。

## 目标

本区帮助你完成第一次接入：安装包、注册组件、引入主题 CSS、跑通 Hello World，再根据项目框架选择导入方式。主题算法、组件 API 和质量基线不会混在这里，它们分别进入 Theme、Components 和 Resources。

## 适用场景

| 场景 | 推荐入口 | 说明 |
| --- | --- | --- |
| 原型、内部工具、文档站 | `@nami-web/ui/register` | 一次注册全部组件，减少配置成本。 |
| 生产业务应用 | 单组件入口 | 只导入当前页面需要的 custom element。 |
| 长期设计系统 | Theme + metadata | 使用 token、anatomy、catalog 和 schema 作为可解释资产。 |

## 阅读路径

<nami-docs-steps>
<ol>
<li><strong>先跑通快速开始。</strong>确认包管理器、主题 CSS、注册入口和第一个组件都能工作。</li>
<li><strong>再选择框架接入方式。</strong>Vue、React 和 Astro 只需要处理事件、类型提示和路由生命周期。</li>
<li><strong>最后进入组件和主题。</strong>组件页查 API，Theme 区调整 token、preset、density、size、radius、contrast 和 motion。</li>
</ol>
</nami-docs-steps>

## 边界

<nami-docs-callout variant="info" title="Start 区不承载 API">
这里不复制组件属性、事件、slots、parts 或 tokens。组件参考由 metadata 驱动，进入 Components；主题导出和算法进入 Theme；质量、i18n、FAQ 和语义结构进入 Resources。
</nami-docs-callout>
