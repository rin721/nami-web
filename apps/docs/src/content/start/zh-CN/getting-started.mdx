# 快速开始

这篇文档按真实接入顺序组织：安装包，加载主题 CSS，注册组件，渲染第一个按钮，再决定全量注册还是按需导入。

## 目标

完成后你应该能在页面中看到一个可交互的 `nami-button`，并知道以后要去哪里查组件 API、主题配置和国际化能力。

## 前置条件

- Node.js 18 或更高版本。
- Vite、Astro、Vue、React 或任意支持 ESM 的构建环境。
- 页面只需要一次主题 CSS；组件可以全量注册，也可以单独导入。

## 安装

<nami-docs-install-tabs></nami-docs-install-tabs>

| 包管理器 | 命令 |
| --- | --- |
| npm | `npm install @nami/ui @nami/themes` |
| pnpm | `pnpm add @nami/ui @nami/themes` |
| yarn | `yarn add @nami/ui @nami/themes` |

## 注册方式

<nami-docs-steps>
<ol>
<li><strong>全量注册。</strong>适合文档站、原型、后台工具和低频页面。</li>
<li><strong>单组件导入。</strong>适合生产页面，让打包器只处理当前页面需要的元素。</li>
<li><strong>副作用注册入口。</strong>需要显式副作用文件时，使用 `@nami/ui/button/register`。</li>
</ol>
</nami-docs-steps>

```ts
// main.ts
import '@nami/themes/default.css';
import '@nami/ui/register';
```

```ts
// page.ts
import '@nami/themes/default.css';
import '@nami/ui/button';
import '@nami/ui/input';
```

```ts
// explicit side-effect register
import '@nami/ui/button/register';
```

## CDN 引用

如果页面不经过 npm 构建，可以使用 `apps/docs/public/cdn/nami-ui/0.1.0/` 下的静态文件。它们会随官网一起发布，并通过 `https://aoi-wen.iqwq.com/cdn/nami-ui/0.1.0/` 访问。

```html
<link rel="stylesheet" href="https://aoi-wen.iqwq.com/cdn/nami-ui/0.1.0/css/default.css" />
<script src="https://aoi-wen.iqwq.com/cdn/nami-ui/0.1.0/nami-ui.global.js"></script>

<nami-button>Hello Nami</nami-button>
```

需要未定义 custom elements 时的首屏 fallback 样式，可以先加载 `critical.css`：

```html
<link rel="stylesheet" href="https://aoi-wen.iqwq.com/cdn/nami-ui/0.1.0/css/critical.css" />
<link rel="stylesheet" href="https://aoi-wen.iqwq.com/cdn/nami-ui/0.1.0/css/default.css" />
<script src="https://aoi-wen.iqwq.com/cdn/nami-ui/0.1.0/nami-ui.global.js"></script>
```

也可以保留 ESM 引用方式：

```html
<link rel="stylesheet" href="https://aoi-wen.iqwq.com/cdn/nami-ui/0.1.0/css/default.css" />
<script type="module" src="https://aoi-wen.iqwq.com/cdn/nami-ui/0.1.0/esm/register.js"></script>
```

按需加载组件时引用 ESM 单组件文件：

```html
<link rel="stylesheet" href="https://aoi-wen.iqwq.com/cdn/nami-ui/0.1.0/css/default.css" />
<script type="module">
  import 'https://aoi-wen.iqwq.com/cdn/nami-ui/0.1.0/esm/components/button.js';
  import 'https://aoi-wen.iqwq.com/cdn/nami-ui/0.1.0/esm/components/input.js';
</script>
```

## 主题定制

`nami-theme` 是运行时主题边界。默认主题 CSS 提供变量，`accent`、`theme`、`density`、`size`、`radius`、`contrast` 和 `motion` 会派生出可复用 token。

```html
<nami-theme accent="#14b8a6" theme="dark" size="lg">
  <nami-button>Scoped action</nami-button>
</nami-theme>
```

<nami-docs-callout variant="info" title="主题不是 React Context">
Nami UI 的核心协议是 CSS variables 与 custom elements。React、Vue 或 Astro 只负责渲染元素，不需要把组件库绑定到某个框架上下文。
</nami-docs-callout>

## 国际化

运行时语言边界由 `nami-config` 提供。文档路由使用 `/zh-CN/` 与 `/en-US/`，组件 fallback 文案使用配置中的 locale。

```html
<nami-config locale="zh-CN">
  <nami-empty></nami-empty>
</nami-config>
```

## 常见错误

| 现象 | 原因 | 修复 |
| --- | --- | --- |
| 元素显示成普通标签 | 没有导入组件或注册入口 | 导入 `@nami/ui/register` 或对应单组件入口。 |
| 样式不对 | 没有加载主题 CSS | 在应用入口导入 `@nami/themes/default.css`。 |
| 主题只影响部分区域 | `nami-theme` 边界放错位置 | 把需要继承变量的组件放在同一个 `nami-theme` 内。 |
