# Overview

Nami UI is a Web Components UI library for applications that need one component source across multiple frontend stacks. The public surface is custom elements, reflected attributes, DOM events, slots, CSS parts, and CSS variables.

## What Lives Here

Start covers project positioning, installation, registration, single-component imports, and framework usage. It is the first-use path for Nami UI, not the place for full component APIs or theme algorithm detail.

## What Does Not Live Here

Component attributes, events, slots, parts, and tokens live in Components. Theme configuration, tokens, algorithms, style presets, Theme Designer, and Theme Lab live in Theme. Quality, i18n, and semantic anatomy contracts live in Resources.

## Where To Go Next

When you are ready to install, continue to [Getting Started](/en-US/docs/getting-started/). When you need to choose controls, open [Components](/en-US/components/). When you need a brand system, open [Theme](/en-US/theme/).

## What ships first

| Area | What it means |
| --- | --- |
| Components | Buttons, chips, inputs, switches, cards, tabs, overlays, shell, and status feedback |
| Themes | Light, dark, custom accent, compact density, reduced motion |
| Skins | Default community UI and an illustration skin powered by the same tokens |
| i18n | `@lit/localize` runtime mode, locale-prefixed docs routes, and localized fallback text |
