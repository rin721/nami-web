# Overview

Nami UI is a Lit and Web Components library. Component behavior, theme tokens, CSS Parts, semantic anatomy, and metadata live in one contract, so the same custom elements work in native HTML, Vue, React, Astro, and other frontend stacks.

## Goal

This area gets your first integration working: install packages, register elements, load theme CSS, render Hello World, then pick a framework path. Theme algorithms, component APIs, and quality baselines live in Theme, Components, and Resources instead of being mixed into Start.

## When To Use Each Entry

| Scenario | Recommended entry | Notes |
| --- | --- | --- |
| Prototypes, internal tools, docs | `@nami/ui/register` | Register every component once and reduce setup work. |
| Production applications | Single component entries | Import only the custom elements used by the page. |
| Long-lived design systems | Theme + metadata | Use tokens, anatomy, catalog, and schemas as explainable assets. |

## Reading Path

<nami-docs-steps>
<ol>
<li><strong>Run Getting Started first.</strong>Confirm the package manager, theme CSS, registration entry, and first component all work.</li>
<li><strong>Pick the framework integration.</strong>Vue, React, and Astro mainly need event, typing, and routing lifecycle glue.</li>
<li><strong>Move to components and theme.</strong>Use component pages for APIs and Theme for tokens, presets, density, size, radius, contrast, and motion.</li>
</ol>
</nami-docs-steps>

<nami-docs-callout variant="info" title="Start does not carry API reference">
