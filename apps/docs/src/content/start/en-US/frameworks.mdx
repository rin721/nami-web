# Frameworks

Nami UI is built from custom elements, so framework integration is not about rewriting components. It is about import location, event handling, type hints, and page lifecycle hooks.

## Goal

Keep the same `nami-*` elements consistent across frameworks: the browser registers components, CSS variables provide styling, and DOM CustomEvent carries interaction data.

## Native HTML

Use this for static pages, low-code previews, CMS fragments, and quick prototypes. Load theme CSS and the global CDN file, then write the tags directly.

```html
<link rel="stylesheet" href="https://nami-web.iqwq.com/cdn/nami-ui/0.1.0/css/default.css" />
<script src="https://nami-web.iqwq.com/cdn/nami-ui/0.1.0/nami-ui.global.js"></script>

<nami-theme accent="#3b82f6">
  <nami-button>Save</nami-button>
</nami-theme>
```

## Vue

Vue should treat `nami-*` as custom elements so the compiler does not resolve them as Vue components.

```ts
// vite.config.ts
import vue from '@vitejs/plugin-vue';

export default {
  plugins: [
    vue({
      template: {
        compilerOptions: {
          isCustomElement: (tag) => tag.startsWith('nami-')
        }
      }
    })
  ]
};
```

When using npm packages, import CSS and components from an entry file or a single-file component:

```vue
<script setup lang="ts">
import '@nami-web/themes/default.css';
import '@nami-web/ui/button';
</script>

<template>
  <nami-button @nami-click="() => console.log('clicked')">Save</nami-button>
</template>
```

When using CDN, put CSS and script tags in `index.html`; Vue templates still render `nami-*` tags directly:

```html
<link rel="stylesheet" href="https://nami-web.iqwq.com/cdn/nami-ui/0.1.0/css/default.css" />
<script src="https://nami-web.iqwq.com/cdn/nami-ui/0.1.0/nami-ui.global.js"></script>
```

```vue
<template>
  <nami-button variant="soft" @nami-click="save">Save</nami-button>
</template>

<script setup lang="ts">
function save(event: CustomEvent) {
  console.log(event.detail);
}
</script>
```

## React

React can render custom elements. For richer events, use `ref` and `addEventListener` to preserve DOM CustomEvent details.

```tsx
import { useEffect, useRef } from 'react';
import '@nami-web/themes/default.css';
import '@nami-web/ui/button';

export function SaveButton() {
  const ref = useRef<HTMLElement>(null);

  useEffect(() => {
    const element = ref.current;
    if (!element) return;
    const onClick = (event: Event) => console.log(event.type);
    element.addEventListener('nami-click', onClick);
    return () => element.removeEventListener('nami-click', onClick);
  }, []);

  return <nami-button ref={ref}>Save</nami-button>;
}
```

<nami-docs-callout variant="warning" title="React type hints">
If TSX reports that `nami-button` does not exist, add custom element JSX declarations to the app or use generated wrappers once they are published.
</nami-docs-callout>

## Astro

Astro pages can import theme CSS and component entries in a layout or entry script. Add page transition elements only when client routing needs them.

```astro
---
import '@nami-web/themes/default.css';
import '@nami-web/ui/register';
---

<nami-config locale="en-US">
  <nami-theme>
    <slot />
  </nami-theme>
</nami-config>
```

## Page Transition Hooks

`nami-top-progress` handles route progress; `nami-page-transition` handles first-paint reveal or blocking tasks. Frameworks only need to call DOM methods in their own lifecycle.

### Astro

```ts
document.addEventListener('astro:before-preparation', () => {
  document.querySelector('nami-top-progress')?.start?.();
});

document.addEventListener('astro:after-swap', () => {
  document.querySelector('nami-top-progress')?.finish?.();
});
```

### Vue Router

```ts
router.beforeEach(() => {
  document.querySelector('nami-top-progress')?.start?.();
});

router.afterEach(() => {
  document.querySelector('nami-top-progress')?.finish?.();
});
```

## Scroll Resistance And Routing

`nami-scroll-smoother` is a page-level scroll controller. Vue Router, Astro, or any client-side router should call `resize()` after a page swap so Lenis can measure the new document height. The component disables itself automatically for reduced motion.

```html
<nami-scroll-smoother preset="balanced" duration="1.2" touch-multiplier="2" anchors stop-inertia-on-navigate></nami-scroll-smoother>
```

```ts
document.querySelector('nami-scroll-smoother')?.applyConfig?.({
  preset: 'gentle',
  duration: 0.9,
  touchMultiplier: 1.6,
  anchors: { offset: -64 }
});
```

### Vue Router

```ts
router.afterEach(() => {
  document.querySelector('nami-top-progress')?.finish?.();
  requestAnimationFrame(() => {
    document.querySelector('nami-scroll-smoother')?.resize?.();
  });
});
```

### Astro

```ts
document.addEventListener('astro:after-swap', () => {
  document.querySelector('nami-top-progress')?.finish?.();
  document.querySelector('nami-scroll-smoother')?.resize?.();
});
```

### Native Async Task

```ts
const transition = document.querySelector('nami-page-transition');
transition?.show?.();
await saveRecord();
transition?.hide?.();
```

## Common Errors

| Symptom | Fix |
| --- | --- |
| Vue reports an unknown component | Configure `isCustomElement`. |
| React event data is missing | Listen with `ref.addEventListener()`. |
| SSR build reports `customElements` is missing | Move component registration to a client-only execution path. |
