# Getting Started

This guide follows the actual integration path: choose npm or CDN, load theme CSS, register components, render the first button, then choose all-component registration or per-component imports.

## Goal

By the end you should see an interactive `nami-button` on the page and know where to find component APIs, theme configuration, and internationalization docs.

## Prerequisites

- Use Node.js 18 or newer when integrating from npm.
- Use the CDN path when you only need to edit HTML.
- Load theme CSS once; register every component or import components one at a time.

## Install

Use npm packages for apps with a build step. Use the docs-site CDN for static pages, CMS snippets, low-code previews, and quick prototypes.

<nami-docs-install-tabs></nami-docs-install-tabs>

| Manager | Command |
| --- | --- |
| npm | `npm install @nami-web/ui @nami-web/themes` |
| pnpm | `pnpm add @nami-web/ui @nami-web/themes` |
| yarn | `yarn add @nami-web/ui @nami-web/themes` |

`@nami-web/*` packages are published to the npm registry. npm and pnpm read the same registry metadata, so a GitHub Actions publish makes the same version available to both package managers.

After installing, import theme CSS and a component entry from the app entry:

```ts
import '@nami-web/themes/default.css';
import '@nami-web/ui/register';
```

CDN usage requires no install step:

```html
<link rel="stylesheet" href="https://nami-web.iqwq.com/cdn/nami-ui/0.1.0/css/default.css" />
<script src="https://nami-web.iqwq.com/cdn/nami-ui/0.1.0/nami-ui.global.js"></script>
```

<nami-docs-callout variant="info" title="Vue / Vite apps">
When using `nami-*` tags in Vue templates, configure `isCustomElement` in the Vite Vue plugin. See the Frameworks guide for the exact config.
</nami-docs-callout>

## Registration Strategy

<nami-docs-steps>
<ol>
<li><strong>Register everything.</strong>Best for docs, prototypes, admin tools, and low-frequency pages.</li>
<li><strong>Import one component.</strong>Best for production pages where the bundler should only touch used elements.</li>
<li><strong>Use explicit side-effect registration.</strong>Use `@nami-web/ui/button/register` when a file should only define the element.</li>
</ol>
</nami-docs-steps>

```ts
// main.ts
import '@nami-web/themes/default.css';
import '@nami-web/ui/register';
```

```ts
// page.ts
import '@nami-web/themes/default.css';
import '@nami-web/ui/button';
import '@nami-web/ui/input';
```

```ts
// explicit side-effect register
import '@nami-web/ui/button/register';
```

## Advanced CDN Usage

CDN assets are published with the docs site and served from `https://nami-web.iqwq.com/cdn/nami-ui/0.1.0/`. The default global file registers every component:

```html
<link rel="stylesheet" href="https://nami-web.iqwq.com/cdn/nami-ui/0.1.0/css/default.css" />
<script src="https://nami-web.iqwq.com/cdn/nami-ui/0.1.0/nami-ui.global.js"></script>

<nami-button>Hello Nami</nami-button>
```

Load `critical.css` first when you want fallback styles before custom elements are defined:

```html
<link rel="stylesheet" href="https://nami-web.iqwq.com/cdn/nami-ui/0.1.0/css/critical.css" />
<link rel="stylesheet" href="https://nami-web.iqwq.com/cdn/nami-ui/0.1.0/css/default.css" />
<script src="https://nami-web.iqwq.com/cdn/nami-ui/0.1.0/nami-ui.global.js"></script>
```

If you prefer native browser ESM, use the ESM register entry:

```html
<link rel="stylesheet" href="https://nami-web.iqwq.com/cdn/nami-ui/0.1.0/css/default.css" />
<script type="module" src="https://nami-web.iqwq.com/cdn/nami-ui/0.1.0/esm/register.js"></script>
```

For per-component loading, import ESM component files directly:

```html
<link rel="stylesheet" href="https://nami-web.iqwq.com/cdn/nami-ui/0.1.0/css/default.css" />
<script type="module">
  import 'https://nami-web.iqwq.com/cdn/nami-ui/0.1.0/esm/components/button.js';
  import 'https://nami-web.iqwq.com/cdn/nami-ui/0.1.0/esm/components/input.js';
</script>
```

## Theme Customization

`nami-theme` is the runtime theme boundary. Theme CSS provides variables; `accent`, `theme`, `density`, `size`, `radius`, `contrast`, and `motion` derive reusable tokens.

```html
<nami-theme accent="#14b8a6" theme="dark" size="lg">
  <nami-button>Scoped action</nami-button>
</nami-theme>
```

<nami-docs-callout variant="info" title="Theme is not React Context">
Nami UI uses CSS variables and custom elements as the core protocol. React, Vue, or Astro render the elements; the library is not bound to a framework context.
</nami-docs-callout>

## Internationalization

`nami-config` provides the runtime locale boundary. Docs routes use `/zh-CN/` and `/en-US/`; component fallback copy reads the configured locale.

```html
<nami-config locale="zh-CN">
  <nami-empty></nami-empty>
</nami-config>
```

## Common Errors

| Symptom | Cause | Fix |
| --- | --- | --- |
| Element renders like a plain tag | Component entry was not imported | Import `@nami-web/ui/register` or the single component entry. |
| Styles look wrong | Theme CSS is missing | Import `@nami-web/themes/default.css` at the app entry. |
| Theme affects only part of the UI | Theme boundary is in the wrong place | Put components that should inherit variables inside the same `nami-theme`. |
