# Theme Engine

`nami-theme` is the theme boundary. It controls the seed accent, theme mode, density, motion, and style preset for all nested components.

## What Lives Here

Theme explains theme boundaries, token layers, derivation algorithms, style presets, Theme Designer, and Theme Lab. This is where users learn how to generate, override, export, and validate a theme.

## What Does Not Live Here

Component APIs do not live in Theme. Each component page owns its parts, states, and component tokens. Site appearance preferences also stay out of Theme; they belong to the Settings utility area.

## Where To Go Next

Read Tokens first to understand the layers, then Theme Algorithm for derivation. Use Theme Designer for direct editing, and Theme Lab for matrix validation.

## Token Layers

| Layer | Examples | Purpose |
| --- | --- | --- |
| Seed | `--nami-accent-50`, `--nami-motion-normal` | Brand and motion inputs |
| Semantic | `--nami-color-primary`, `--nami-surface`, `--nami-text` | Shared meaning |
| Component | `--nami-button-bg`, `--nami-chip-selected-bg` | Per-component control |

Interactive states are expected to follow the same accent seed: hover, selected, focus, switch checked, and loading indicators.
