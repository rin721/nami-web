# Theme Overview

Nami UI's theme system is not a pile of unrelated CSS variables. It is a deterministic chain: a small seed input derives reference, semantic, component, and style tokens, then CSS variables apply them to components.

## Goal

After reading this page you should understand what `nami-theme` owns, why tokens are layered, which options are safe at runtime, and when to use Theme Designer or Theme Lab.

## Seed Model

| Seed | Values | Impact |
| --- | --- | --- |
| `accent` | Any brand color | Primary scale, focus ring, interactive emphasis. |
| `theme` | `light` / `dark` / `system` | Surface, text, and border semantic colors. |
| `density` | `comfortable` / `compact` | Component spacing and page density. |
| `size` | `sm` / `md` / `lg` | Default control height, padding, font size, and icon size. |
| `radius` | `sharp` / `soft` / `round` | Control, card, and overlay radius scale. |
| `contrast` | `normal` / `high` | Text, border, and focus contrast. |
| `motion` | `normal` / `reduced` | Durations, easing, and loading motion. |
| `stylePreset` | `default` / `illustration` | Product UI style or illustration style. |

`nami-theme` can be nested. The inner boundary overrides outer variables while unspecified tokens continue to inherit. A component's explicit `size` attribute has priority over global theme size.

## Token Layers

<nami-docs-steps>
<ol>
<li><strong>Reference tokens.</strong>Raw values such as scales, spacing, type, and shadows.</li>
<li><strong>Semantic tokens.</strong>Cross-component meaning such as `--nami-surface`, `--nami-text`, and `--nami-color-primary`.</li>
<li><strong>Component tokens.</strong>Slot-level variables such as `--nami-button-bg` and `--nami-input-border`.</li>
<li><strong>Recipes and slot recipes.</strong>Map variants, states, and parts into stable styling contracts.</li>
</ol>
</nami-docs-steps>

## When To Use Tools

| Tool | Use case |
| --- | --- |
| Theme Designer | Tune seeds, export CSS, DTCG JSON, TS config, and inspect diagnostics plus affected components. |
| Theme Lab | Check light/dark, compact/comfortable, normal/high contrast, and default/illustration combinations. |
| Settings Config | Export the current docs appearance as copyable configuration. |
