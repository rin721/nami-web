# Theme Algorithm

The theme algorithm turns a small input into a complete visual system. It must be predictable, exportable, and diagnosable so themes do not drift between the designer and production.

## Goal

Understand the roles of `deriveNamiTheme(seed)`, `defineNamiTheme(config)`, and `createNamiThemeSystem(config)`, then export the result to CSS, DTCG, and Theme Studio.

## Derivation Flow

<nami-docs-steps>
<ol>
<li><strong>Validate seeds.</strong>Invalid enum values produce diagnostics and fall back to defaults.</li>
<li><strong>Generate palette.</strong>`accent` derives the primary scale; `theme` maps light/dark semantics.</li>
<li><strong>Apply modes.</strong>`density`, `size`, `radius`, `contrast`, and `motion` inject layout, control, and motion variables.</li>
<li><strong>Merge overrides.</strong>`tokens`, `semanticTokens`, `components`, `recipes`, and `slotRecipes` apply after base derivation.</li>
<li><strong>Emit artifacts.</strong>Return CSS variables, DTCG JSON, TS config, diagnostics, and a token tree.</li>
</ol>
</nami-docs-steps>

## Basic Code

```ts
import { defineNamiTheme, createNamiThemeSystem } from '@nami-web/tokens/theme';

const config = defineNamiTheme({
  seed: {
    accent: '#14b8a6',
    theme: 'dark',
    stylePreset: 'illustration',
    density: 'compact',
    size: 'md',
    radius: 'soft',
    contrast: 'high',
    motion: 'normal'
  },
  components: {
    'nami-button': {
      primary: {
        bg: 'var(--nami-color-primary)'
      }
    }
  }
});

const system = createNamiThemeSystem(config);
```

## Export

```ts
import { themeToCssVars, themeToCssText, themeToDtcg } from '@nami-web/tokens/theme';
import { createNamiThemeStudio } from '@nami-web/tokens/theme-studio';

const cssVars = themeToCssVars(system);
const cssText = themeToCssText(system, '.brand-theme');
const dtcg = themeToDtcg(system);
const studio = createNamiThemeStudio(config, { selector: '.brand-theme' });
```

`createNamiThemeStudio()` also returns CSS, DTCG JSON, TS config, contrast diagnostics, affected components, and the derived token tree. Theme Designer and Settings Config both use this shared capability.

## Common Errors

| Error | Fix |
| --- | --- |
| Writing component-private values as seeds | Put them in `components` or scoped CSS overrides. |
| Copying temporary designer code into apps | Export TS config or scoped CSS from Theme Designer. |
| Forgetting high contrast | Check normal/high contrast combinations in Theme Lab. |
