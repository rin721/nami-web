# Tokens

Nami UI components do not read seed values directly. Seeds generate tokens; components consume semantic, component, and style tokens so themes can change while component behavior stays stable.

## Goal

Learn which variable to override: application-wide meaning usually changes semantic tokens; component appearance changes component tokens; presets change style tokens; raw values and algorithm input stay in theme config.

## Layers

| Layer | Examples | Responsibility |
| --- | --- | --- |
| Reference | `--nami-accent-50`, `--nami-space-3` | Raw scales, spacing, type, and shadows. |
| Semantic | `--nami-color-primary`, `--nami-surface`, `--nami-text` | Shared cross-component meaning. |
| Component | `--nami-button-bg`, `--nami-checkbox-bg`, `--nami-progress-fill-bg` | Single component or slot overrides. |
| Style | `--nami-style-stroke-width`, `--nami-style-background-pattern` | Visual language from style presets. |

## Rules

<nami-docs-steps>
<ol>
<li><strong>Change seeds first.</strong>Brand color, scheme, density, size, radius, contrast, and motion should start in `NamiThemeSeed`.</li>
<li><strong>Then change semantic tokens.</strong>Override variables such as `--nami-surface` or `--nami-text` for global meaning.</li>
<li><strong>Finally change component tokens.</strong>Override a component token when only one component slot should change.</li>
</ol>
</nami-docs-steps>

```css
.admin-theme {
  --nami-color-primary: #0f766e;
  --nami-button-bg: #0f766e;
  --nami-button-text: white;
}
```

## TS And DTCG

```ts
import { defineNamiTheme, createNamiThemeSystem, themeToCssText, themeToDtcg } from '@nami/tokens/theme';

const config = defineNamiTheme({
  seed: {
    accent: '#14b8a6',
    theme: 'dark',
    density: 'compact',
    size: 'lg',
    radius: 'soft',
    contrast: 'high'
  }
});

const system = createNamiThemeSystem(config);
const css = themeToCssText(system, '.admin-theme');
const dtcg = themeToDtcg(system);
```

## Common Errors

| Error | Why it is risky | Better path |
| --- | --- | --- |
| Component internals read seeds directly | Seeds are algorithm input, not stable slots | Use semantic or component tokens. |
| Every page hand-writes variables | Hard to diagnose and export | Use `defineNamiTheme()` and Theme Designer. |
| Color changes skip contrast checks | Dark and high-contrast modes can fail | Use Theme Studio diagnostics. |
