# Tokens

Nami UI keeps theme customization behind CSS Variables and a deterministic theme system. Components consume semantic, component, and style tokens instead of scattered seed values.

## Layers

| Layer | Examples | Responsibility |
| --- | --- | --- |
| Seed | `--nami-accent-50`, `--nami-layout-gutter` | Brand, density, layout, and motion inputs |
| Semantic | `--nami-color-primary`, `--nami-surface`, `--nami-text` | Shared component meaning |
| Component | `--nami-button-bg`, `--nami-checkbox-bg`, `--nami-progress-fill-bg` | Component or slot overrides |
| Style | `--nami-style-stroke-width`, `--nami-style-background-pattern` | Visual language for style presets |

## Boundary

Application code should override semantic or component tokens first. To generate a full theme, use `defineNamiTheme()` and `createNamiThemeSystem()` to emit scoped CSS.
