# Style Presets

A preset is a visual language, not a second component implementation. `default` and `illustration` share the same `nami-*` elements, attributes, events, slots, parts, and component tokens.

## Goal

Know when to use the default style, when to use illustration style, and how to keep preset switching from breaking component APIs, accessibility, or theme exports.

| Preset | Best for | Visual strategy |
| --- | --- | --- |
| `default` | SaaS, admin, CRM, configuration panels | Quiet, dense, focused on information hierarchy and repeated workflows. |
| `illustration` | Websites, story pages, brand showcases, tutorials | Stronger strokes, offset shadows, paper texture, and decorative patterns. |

## Preset API

```html
<nami-theme style-preset="illustration">
  <nami-card>
    <nami-button>Primary action</nami-button>
  </nami-card>
</nami-theme>
```

```ts
import { defineNamiTheme } from '@nami/tokens/theme';

export default defineNamiTheme({
  seed: {
    stylePreset: 'illustration',
    radius: 'soft'
  }
});
```

## Design Rules

<nami-docs-steps>
<ol>
<li><strong>Do not change component semantics.</strong>A button stays a button; a dialog still manages focus.</li>
<li><strong>Do not duplicate component code.</strong>Preset differences belong in style tokens, CSS Parts, and recipe output.</li>
<li><strong>Test every preset in the matrix.</strong>Cover light/dark, density, size, contrast, and motion.</li>
</ol>
</nami-docs-steps>

## Common Errors

| Error | Fix |
| --- | --- |
| Writing illustration style as component branches | Move differences to style tokens or CSS Parts. |
| Using illustration broadly in operational tools | Keep `default`; use illustration locally for empty states or onboarding. |
| Checking only light mode | Use Theme Lab for dark and high contrast. |
