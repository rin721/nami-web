# 主题算法

主题算法把少量输入变成完整视觉系统。它必须可预测、可导出、可诊断，避免“设计器里好看，业务里失控”的主题漂移。

## 目标

理解 `deriveNamiTheme(seed)`、`defineNamiTheme(config)` 和 `createNamiThemeSystem(config)` 的分工，并知道如何把输出接入 CSS、DTCG 和官网 Theme Studio。

## 派生流程

<nami-docs-steps>
<ol>
<li><strong>校验 seed。</strong>非法枚举会产生 diagnostic，并回退到默认值。</li>
<li><strong>生成 palette。</strong>`accent` 派生主色阶，`theme` 决定 light/dark 语义映射。</li>
<li><strong>应用模式。</strong>`density`、`size`、`radius`、`contrast` 和 `motion` 注入布局、控件和动效变量。</li>
<li><strong>合并覆盖。</strong>`tokens`、`semanticTokens`、`components`、`recipes` 与 `slotRecipes` 作为后置覆盖。</li>
<li><strong>输出产物。</strong>生成 CSS variables、DTCG JSON、TS config、诊断和 token tree。</li>
</ol>
</nami-docs-steps>

## 基础代码

```ts
import { defineNamiTheme, createNamiThemeSystem } from '@nami-web/tokens/theme';

const config = defineNamiTheme({
  seed: {
    accent: '#14b8a6',
    theme: 'dark',
    stylePreset: 'illustration',
    density: 'compact',
    size: 'md',
    radius: 'soft',
    contrast: 'high',
    motion: 'normal'
  },
  components: {
    'nami-button': {
      primary: {
        bg: 'var(--nami-color-primary)'
      }
    }
  }
});

const system = createNamiThemeSystem(config);
```

## 导出

```ts
import { themeToCssVars, themeToCssText, themeToDtcg } from '@nami-web/tokens/theme';
import { createNamiThemeStudio } from '@nami-web/tokens/theme-studio';

const cssVars = themeToCssVars(system);
const cssText = themeToCssText(system, '.brand-theme');
const dtcg = themeToDtcg(system);
const studio = createNamiThemeStudio(config, { selector: '.brand-theme' });
```

`createNamiThemeStudio()` 会额外返回 CSS、DTCG JSON、TS config、contrast 诊断、受影响组件和派生 token 树。官网 Theme Designer 和 Settings Config 都使用这一公共能力。

## 常见错误

| 错误 | 修复 |
| --- | --- |
| 把组件私有变量写成 seed | 把它放进 `components` 或 CSS scoped override。 |
| 在业务里复制设计器临时代码 | 从 Theme Designer 导出 TS config 或 scoped CSS。 |
| 忘记 high contrast | 在 Theme Lab 检查 normal/high contrast 组合。 |
