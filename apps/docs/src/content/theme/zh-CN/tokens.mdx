# Tokens

Nami UI 将换肤边界集中在 CSS Variables 和确定性主题系统之后。组件内部只消费 semantic、component 和 style tokens，不直接依赖零散 seed。

## 分层

| 层级 | 示例 | 责任 |
| --- | --- | --- |
| Seed | `--nami-accent-50`, `--nami-layout-gutter` | 品牌、密度、布局和动效输入 |
| Semantic | `--nami-color-primary`, `--nami-surface`, `--nami-text` | 组件共享语义 |
| Component | `--nami-button-bg`, `--nami-checkbox-bg`, `--nami-progress-fill-bg` | 单组件或槽位覆盖 |
| Style | `--nami-style-stroke-width`, `--nami-style-background-pattern` | 风格 preset 的视觉语言 |

## 使用边界

应用层优先覆盖 semantic 或 component token。需要生成完整主题时，使用 `defineNamiTheme()` 和 `createNamiThemeSystem()` 输出 scoped CSS。
