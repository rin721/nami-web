# Tokens

Nami UI 组件内部不直接依赖 seed。seed 只负责生成 token，组件消费 semantic token、component token 和 style token，这样主题可以替换，组件行为仍然稳定。

## 目标

学会判断应该覆盖哪个变量：应用层通常改 semantic token；组件外观改 component token；风格 preset 改 style token；原始数值和算法输入留给 theme config。

## 分层

| 层级 | 示例 | 责任 |
| --- | --- | --- |
| Reference | `--nami-accent-50`、`--nami-space-3` | 原始色阶、空间、字号、阴影。 |
| Semantic | `--nami-color-primary`、`--nami-surface`、`--nami-text` | 跨组件共享意义。 |
| Component | `--nami-button-bg`、`--nami-checkbox-bg`、`--nami-progress-fill-bg` | 单组件或槽位覆盖。 |
| Style | `--nami-style-stroke-width`、`--nami-style-background-pattern` | 风格 preset 的视觉语言。 |

## 使用规则

<nami-docs-steps>
<ol>
<li><strong>先改 seed。</strong>品牌色、明暗、密度、尺寸、圆角、对比度和动效优先走 `NamiThemeSeed`。</li>
<li><strong>再改 semantic。</strong>需要全局语义变化时，覆盖 `--nami-surface` 或 `--nami-text` 这类变量。</li>
<li><strong>最后改 component。</strong>只想改某个组件槽位时，覆盖对应 component token。</li>
</ol>
</nami-docs-steps>

.admin-theme {

  --nami-button-bg: #0f766e;
  --nami-button-text: white;

## TS 与 DTCG

```ts
import { defineNamiTheme, createNamiThemeSystem, themeToCssText, themeToDtcg } from '@nami/tokens/theme';

const config = defineNamiTheme({
  seed: {
    accent: '#14b8a6',
    theme: 'dark',
    density: 'compact',
    size: 'lg',
    radius: 'soft',
    contrast: 'high'
  }
});

const system = createNamiThemeSystem(config);
const css = themeToCssText(system, '.admin-theme');
const dtcg = themeToDtcg(system);
```

## 常见错误

| 错误 | 为什么不推荐 | 推荐方式 |
| --- | --- | --- |
| 组件内部直接消费 seed | seed 是算法输入，不是稳定槽位 | 使用 semantic 或 component token。 |
| 每个页面手写一套变量 | 难以诊断、难以导出 | 使用 `defineNamiTheme()` 和 Theme Designer。 |
| 只改颜色不测对比度 | 暗色和高对比模式可能失败 | 使用 Theme Studio 诊断。 |
