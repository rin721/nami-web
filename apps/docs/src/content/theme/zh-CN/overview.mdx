# 主题概览

Nami UI 的主题系统不是一组散落 CSS 变量，而是一条确定的派生链：少量 seed 输入，经过算法生成 reference、semantic、component 和 style token，最后通过 CSS variables 作用到组件。

## 目标

读完后你应该知道 `nami-theme` 负责什么、token 为什么分层、哪些能力适合运行时调整，以及什么时候应该进入 Theme Designer 或 Theme Lab。

## Seed 模型

| Seed | 可选值 | 影响 |
| --- | --- | --- |
| `accent` | 任意品牌色 | 主色阶、focus ring、交互强调色。 |
| `theme` | `light` / `dark` / `system` | 表面、文本和边框语义色。 |
| `density` | `comfortable` / `compact` | 组件间距和页面密度。 |
| `size` | `sm` / `md` / `lg` | 默认控件高度、内边距、字号和图标尺寸。 |
| `radius` | `sharp` / `soft` / `round` | 控件、卡片、弹层的圆角层级。 |
| `contrast` | `normal` / `high` | 文本、边框和焦点对比度。 |
| `motion` | `normal` / `reduced` | 过渡时长、easing 和加载动效。 |
| `stylePreset` | `default` / `illustration` | 默认产品风格或插画风格。 |

`nami-theme` 可以嵌套。内部边界覆盖外部变量，没有显式配置的 token 继续继承。组件自身显式 `size` 属性优先于全局 theme size。

## Token 层级

<nami-docs-steps>
<ol>
<li><strong>Reference token。</strong>色阶、空间、字号、阴影等原始值。</li>
<li><strong>Semantic token。</strong>`--nami-surface`、`--nami-text`、`--nami-color-primary` 这类跨组件意义。</li>
<li><strong>Component token。</strong>`--nami-button-bg`、`--nami-input-border` 这类组件槽位变量。</li>
<li><strong>Recipe 与 slot recipe。</strong>把 variant、state、part 映射到稳定样式契约。</li>
</ol>
</nami-docs-steps>

## 什么时候用工具

| 工具 | 适用场景 |
| --- | --- |
| Theme Designer | 调 seed、导出 CSS、DTCG JSON、TS config，查看诊断和受影响组件。 |
| Theme Lab | 批量检查 light/dark、compact/comfortable、normal/high contrast、default/illustration。 |
| Settings Config | 在官网里把当前外观导出成可复制配置。 |
