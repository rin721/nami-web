# 风格 Preset

Preset 是视觉语言，不是第二套组件实现。`default` 与 `illustration` 共享同一批 `nami-*` 元素、属性、事件、slots、parts 和 component token。

## 目标

知道什么时候使用默认风格，什么时候使用插画风格，以及如何确保 preset 切换不会破坏组件 API、可访问性和主题导出。

| Preset | 适用场景 | 视觉策略 |
| --- | --- | --- |
| `default` | SaaS、后台、CRM、配置面板 | 安静、密集、强调信息层级和重复操作效率。 |
| `illustration` | 官网、故事化页面、品牌展示、教学内容 | 更明显的描边、偏移阴影、纸张质感和装饰图案。 |

## Preset API

```html
<nami-theme style-preset="illustration">
  <nami-card>
    <nami-button>主要操作</nami-button>
  </nami-card>
</nami-theme>
```

```ts
import { defineNamiTheme } from '@nami-web/tokens/theme';

export default defineNamiTheme({
  seed: {
    stylePreset: 'illustration',
    radius: 'soft'
  }
});
```

## 设计规则

<nami-docs-steps>
<ol>
<li><strong>不改变组件语义。</strong>button 仍是 button，dialog 仍承担焦点管理。</li>
<li><strong>不复制组件代码。</strong>preset 只能改变 token、parts 和 recipe 输出。</li>
<li><strong>必须进入矩阵测试。</strong>每个 preset 都要覆盖 light/dark、density、size、contrast 和 motion。</li>
</ol>
</nami-docs-steps>

## 常见错误

| 错误 | 修复 |
| --- | --- |
| 把插画风格写成组件分支 | 把差异放进 style token 或 CSS Parts。 |
| 在后台工具里大面积使用插画 preset | 保留 default，必要时只在空状态或引导区局部使用。 |
| 只看 light 模式 | 在 Theme Lab 检查 dark 和 high contrast。 |
