## Why Did A Component Not Render?

The component is usually not registered.

```ts
import '@nami/themes/default.css';
import '@nami/ui/register';
```

Or import a single component:

```ts
import '@nami/themes/default.css';
import '@nami/ui/button';
```

## Why Do Styles Look Wrong?

Check three things first:

<nami-docs-steps>
<ol>
<li><strong>Theme CSS is imported.</strong>The app entry should import `@nami/themes/default.css`.</li>
<li><strong>The theme boundary wraps the component.</strong>Local themes need components inside the same `nami-theme`.</li>
<li><strong>Hard-coded values bypass tokens.</strong>Color, radius, and motion should usually come from seeds or CSS variables.</li>
</ol>
</nami-docs-steps>

## Why Did Language Not Switch?

Docs route locale and component runtime locale are separate. The page path chooses `/zh-CN/` or `/en-US/`; component fallback copy reads `nami-config locale`.

```html
<nami-config locale="en-US">
  <nami-empty></nami-empty>
</nami-config>
```

## Why Is The Code Example Showing The Wrong Framework?

Open Settings and choose HTML, Vue, or React in Preferences. Component detail pages use that preference to show the default snippet.
