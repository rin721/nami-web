# Internationalization

Internationalization has two layers: docs content routes and component runtime copy. Docs use `/zh-CN/` and `/en-US/` prefixes; components read fallback copy from the locale on `nami-config`.

## Goal

Keep docs structure aligned across languages, let component copy switch at runtime, and avoid separate language logic in content pages, component pages, and settings pages.

## Runtime Boundary


## Route Rules

| Location | Rule |
| --- | --- |
| Home | `/zh-CN/` and `/en-US/`. |
| Start | `/docs/[slug]/`; slugs stay aligned across languages. |
| Theme | `/theme/[slug]/`; tool pages also belong to Theme. |
| Resources | `/resources/[slug]/`; do not reuse `/docs/*`. |
| Components | `/components/[slug]/`; component slugs are not translated. |

## Content Sync

<nami-docs-steps>
<ol>
<li><strong>Keep heading structure aligned.</strong>Core sections should have the same order and count.</li>
<li><strong>Keep examples semantically aligned.</strong>Code APIs can stay English, but explanation copy should match the current language.</li>
<li><strong>Keep routes aligned.</strong>Add new pages to the docs structure registry and static path generation at the same time.</li>
</ol>
</nami-docs-steps>

## Component Copy

Component fallback copy should read from runtime locale first. When product copy needs to be more specific, use component attributes, slots, or custom content instead of project-specific language branches inside components.

## Common Errors

| Symptom | Fix |
| --- | --- |
| Language switch returns to home | Use `switchLocalePath()` to keep the current logical path. |
| Component fallback still appears in English | Check that the page is inside `nami-config locale="en-US"` or the intended locale. |
| Chinese and English docs have different sections | Add the matching MDX page and update browser tests. |
