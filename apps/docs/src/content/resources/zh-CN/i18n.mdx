# 国际化

国际化分为两层：官网内容路由和组件运行时文案。官网使用 `/zh-CN/` 与 `/en-US/` 前缀；组件使用 `nami-config` 的 locale 读取 fallback 文案。

## 目标

让文档结构双语一致，让组件文案可以在运行时切换，同时避免内容页、组件页和设置页各自维护不同语言逻辑。

## 运行时边界

## 路由规则

| 位置 | 规则 |
| --- | --- |
| 首页 | `/zh-CN/` 与 `/en-US/`。 |
| Start | `/docs/[slug]/` 两种语言 slug 保持一致。 |
| Theme | `/theme/[slug]/`，工具页也属于 Theme。 |
| Resources | `/resources/[slug]/`，不复用 `/docs/*`。 |
| Components | `/components/[slug]/`，组件 slug 不翻译。 |

## 内容同步

<nami-docs-steps>
<ol>
<li><strong>标题结构同步。</strong>中英文页面的核心章节数量和顺序必须一致。</li>
<li><strong>示例语义同步。</strong>代码可以保留英文 API，但说明文案要匹配当前语言。</li>
<li><strong>路由同步。</strong>新增页面时同时加入 docs structure registry 和静态路径生成。</li>
</ol>
</nami-docs-steps>

## 组件文案

组件默认文案应优先来自 runtime locale。业务需要更具体文案时，使用组件属性、slot 或自定义内容覆盖，不要在组件内部新增项目专属语言分支。

## 常见错误

| 现象 | 修复 |
| --- | --- |
| 切换语言后回到首页 | 使用 `switchLocalePath()` 保留当前 logical path。 |
| 组件 fallback 仍是英文 | 检查页面是否包在 `nami-config locale="zh-CN"` 内。 |
| 中英文文档章节不一致 | 补齐对应语言 MDX，并更新浏览器测试。 |
