# 语义结构

语义结构让组件可以被解释、被测试、被主题工具消费。它不是视觉注释，而是组件公开契约的一部分。

## 目标

让每个交互组件都能稳定暴露 host 状态、CSS Parts、tokens、slots、events 和 anatomy，从而支持文档生成、主题 DIY、视觉回归和跨框架 wrapper。

## Anatomy

Anatomy 描述组件内部可解释部位，例如 `base`、`control`、`label`、`description`、`error`、`icon`、`content`。组件页的 Anatomy 章节来自公共 metadata，不在 MDX 里手写。

## 状态契约

| 状态 | 用途 |
| --- | --- |
| `data-state` | open/closed、checked/unchecked、active/inactive 等互斥状态。 |
| `data-disabled` | 禁用样式和事件阻断。 |
| `data-invalid` | 表单校验失败。 |
| `data-loading` | 阻断重复提交并暴露加载状态。 |
| `data-placement` | tooltip、popover、drawer 等浮层位置。 |

## CSS Parts

CSS Parts 是深度 DIY 的安全边界。组件内部可以重构 DOM，但只要公开 part 名称不变，用户样式和主题工具就能继续工作。

```css
nami-input::part(control) {
  border-color: var(--nami-color-primary);
}
```

## Style Hooks

Style hook 把 part、state 和 tokens 连起来，例如 `control:invalid` 对应错误边框 token。它比“看 DOM 猜样式”稳定，也更适合自动生成主题设计器。
